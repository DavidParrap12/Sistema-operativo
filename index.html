¿Qué es la gestión de E/S?

La interfaz crítica entre la CPU y el mundo exterior.

input

Concepto de Entrada

Flujo de datos desde dispositivos periféricos (teclado, red, sensores) hacia la memoria del sistema para su procesamiento por la CPU📌 CPUUnidad Central de Procesamiento. Ejecuta instrucciones y coordina todos los componentes del sistema..

output

Concepto de Salida

Envío de datos procesados desde la memoria hacia el exterior (monitores, impresoras, almacenamiento externo o redes).

Funciones principales

tune

Control

Envío de comandos a los dispositivos para iniciar operaciones de lectura o escritura mediante registros de control📌 Registros de ControlPosiciones de memoria mapeadas que permiten a la CPU enviar instrucciones directamente al controlador del dispositivo..

hourglass_top

Buffering

Uso de memorias intermedias (buffers📌 BufferZona de memoria temporal que almacena datos en tránsito para compensar diferencias de velocidad entre productor y consumidor.) para compensar las diferencias de velocidad entre dispositivos.

print

Spooling

Almacenamiento temporal (spooling📌 SpoolingSimultaneous Peripheral Operations On-Line. Cola de trabajos en disco que permite compartir dispositivos secuenciales (ej: impresoras) entre múltiples procesos.) para dispositivos compartidos de acceso secuencial como impresoras.

bolt

Interrupciones

Manejo de Eventos

Gestión de señales asíncronas (IRQ📌 IRQInterrupt Request. Línea de señal de hardware que indica a la CPU que un dispositivo necesita atención inmediata.) que notifican la finalización de una tarea de hardware.

report

Errores

Detección y corrección de fallos en la transferencia de datos para garantizar la integridad.

apps Aplicación de Usuario expand_more

Los programas del usuario solicitan operaciones de E/S mediante llamadas al sistema (read(), write()). No interactúan directamente con el hardware; usan una API abstracta proporcionada por el kernel.

memory Sistema Operativo (Kernel) expand_more

El kernel gestiona las colas de solicitudes, planifica las operaciones de E/S, maneja interrupciones y aplica políticas de buffering y caché. Es el intermediario central entre aplicaciones y drivers.

developer_mode Drivers (Controladores) expand_more

Traducen los comandos genéricos del OS en instrucciones específicas del hardware. Cada dispositivo (GPU, SSD, NIC) tiene su propio driver que conoce los registros y protocolos del controlador.

developer_board Hardware (Dispositivo Físico) expand_more

El controlador del dispositivo ejecuta la operación física: mover el cabezal del disco, encender un LED, transmitir un paquete por la red. Comunica su estado al driver vía interrupciones o DMA.

Estrategias de Transferencia

sync Síncrona

La CPU espera bloqueada (usualmente haciendo Polling📌 PollingTécnica donde la CPU verifica repetidamente (en bucle) el estado del dispositivo. Simple pero consume muchos ciclos de CPU.) hasta que el dispositivo completa la operación. Fácil de implementar pero ineficiente para multitarea.

sync_alt Asíncrona

La CPU inicia la operación y continúa ejecutando otros procesos (delegando con DMA📌 DMADirect Memory Access. Permite a los dispositivos transferir datos directamente a/desde la memoria RAM sin intervención de la CPU, liberándola para otras tareas.). Es notificada mediante interrupciones al terminar.

Técnica Descripción Carga CPU Uso Ideal
Polling Consulta constante del estado del dispositivo. Muy Alta Dispositivos ultra-rápidos.
Interrupciones El dispositivo avisa cuando está listo. Media Teclados, ratones, redes.
DMA Transferencia directa Memoria-Dispositivo. Baja Discos duros, Video 4K.
